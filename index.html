🎮 遊戲概覽

3D Dino 是以 Three.js 建構的 3D 體素風格恐龍跑酷遊戲， 靈感源自 Chrome 離線小恐龍，但加入了完整的三維場景、 動態光影、粒子特效，以及透過 MQTT / WebRTC DataChannel 接收體感控制指令。

遊戲嵌入在更大的「遊戲中心（index.html）」中，是四款遊戲之一， 與「憤怒火箭」、「彈珠彈射」、「AI 聊天室」並列於選單。

GameManager.loop() {
  if (!this.isPlaying) return;     // 停止條件

  requestAnimationFrame(() => game.loop());
  this.render();
}

GameManager.render() {
  let timeDelta = clock.getDelta();
  if (timeDelta > 0.15) timeDelta = 0.15; // 夾住防止大步跳躍

  player.update(timeDelta);
  enemy.update(timeDelta);
  nature.update(timeDelta);
  input.update();
  effects.update(timeDelta);
  nebulaSystem.update();

  renderer.render(scene, camera);
  score.update(timeDelta);
}

⚙️ 各管理器功能

LoadManager — 資產依賴樹

所有 .vox 模型、貼圖都透過 LoadManager 管理， 支援依賴關係（dep）：若資產 A 需要等資產 B 載入後才能處理， 設定 deps 陣列即可自動等待。 

// 設定資產載入器（含依賴）
load_manager.set_loader('cactus', ['ground'], () => {
  // 等 ground 載入完後才執行此 callback
  loadVoxModel('dino3d/objects/cactus/cactus.vox', mesh => {
    load_manager.set_vox('cactus', mesh);
    load_manager.set_status('cactus', true);
  });
});

// 一次載入全部（含進度回調）
load_manager.load_all(
  () => { /* 全部完成 */ },
  () => { let p = load_manager.getLoadPercentage(); }
);

🦖 玩家物理系統

玩家（T-Rex）使用速度 + 重力積分模擬跳躍， 每幀根據 timeDelta 更新 Y 座標： 

// 跳躍物理（每幀執行）
frame.position.y += frame.vel * timeDelta;
frame.vel        += frame.gravity * timeDelta;   // gravity 為負值

// 落地判斷
if (frame.position.y <= frame.init_y) {
  jump.is_active = false;
  frame.position.y = frame.init_y;   // 強制修正至地面
}

跳躍增強（Boost）

按住空白鍵超過 0.20 秒後，跳躍速度乘以 boost.vel，重力切換為 boost.gravity， 實現「長按飛得更高」效果： 

if (!frame.boost && keys.space.down && spaceHeld > 0.20) {
  frame.vel     = frame.vel * jump.boost.vel;    // × 1.1 ~ 1.5（依速度等級）
  frame.gravity = jump.boost.gravity;            // -30 ~ -70（依速度等級）
  frame.boost   = true;
}

快速下墜（Fast Fall）

跳躍中按下方向鍵（Down/Ctrl），重力每幀乘以 1.1，快速墜落並改用蜷縮（band）幀動畫：

if (keys.down.down && !block_fall_fast) {
  frame.gravity *= 1.1;              // 加速下墜
  frame.geometry = frames_band[...]; // 切換至蜷縮姿勢
  collisionBox.scale.y = 0.5;       // 碰撞箱縮小
}

碰撞箱（CollisionBox）

碰撞使用不可見的 THREE.BoxGeometry(0.5, 1.7, 0.7)， 依姿勢動態調整：

狀態scale.yscale.zpos.y 偏移
站立1.01.0+ 1.4
蜷縮（地面）0.52.5+ 0.7
快速下墜0.52.5+ 0.8（更低）

動畫幀切換速度

奔跑動畫速度與遊戲速度同步，速度越快恐龍腳步越快： 

// anim_speed 隨速度動態計算
this.anim_speed = 0.18 / (enemy.config.vel / 2);

仙人掌群組（Cactus Group）

每組仙人掌由 1 個主體 + 最多 2 個「尾隨」（tail）組成， 尾隨仙人掌體積較小，緊靠在主體後方： 

// 100% 機率生成第一個尾隨
// 25% 機率再生成第二個尾隨
if (Math.random() * 100 < 100) enemiesGroup.push(createEnemy('cactus', true, 0));
if (Math.random() * 100 < 25)  enemiesGroup.push(createEnemy('cactus', true, 1));

翼龍（Ptero）特性

  • 分數達 400 時開始生成
  • 隨機出現在三個高度（地面 / +1.3 / +3.5）
  • Z 位置 > ptero_z_speedup = -35 時速度提升 1.7 倍（快速逼近）
  • 每 0.10 秒切換一幀動畫（共 6 幀拍翅）

碰撞偵測

// 使用 Three.js Box3 AABB 碰撞
let eBox = new THREE.Box3().setFromObject(enemy[j]);
let pBox = new THREE.Box3().setFromObject(player.collisionBox);

if (eBox.intersectsBox(pBox) && enemy[j].visible) {
  game.stop(); // 碰撞 → 遊戲結束
}

📊 分數與等級系統

分數以 add_vel × timeDelta 每幀疊加，每 100 分觸發一次 flash()（加速 + 音效 + 分數閃爍）。 最高分儲存於 localStorage。

0
開始 — 得分 400
add_vel = 10 分/秒，只有仙人掌，每 100 分加速一次
vel: 10/s
1
分數 400 — 翼龍登場
enemy.spawnPteros() — 將 15 隻翼龍加入 Pool，開始混入仙人掌
vel: 10/s
2
分數 1000 — 得分加速
add_vel 提升為 20/s，分數累積速度翻倍
vel: 20/s
3
分數 3000 — 高速模式
add_vel 提升為 40/s，遊戲速度趨近上限 35
vel: 40/s

速度等級表

每次 flash() 調用 enemy.increase_velocity(1)，跳躍物理同步調整：

速度等級enemy.vel 範圍跳躍初速 (vel)重力 (gravity)Boost 乘數粒子速度
Level 1 < 10 15 -37 ×1.1 / -30g spd=4（預設）
Level 2 10 ~ 20 19 -60 ×1.1 / -40g spd=7
Level 3 20 ~ 30 25 -100 ×1.3 / -70g spd=10
Level 4 ≥ 30（上限 35） 30 -150 ×1.5 / -70g 停止粒子

enemy.vel 上限為 35（increase_velocity 中有守衛）。 開局時呼叫 enemy.increase_velocity(15, true) 直接設定初始速度，跳過加速邏輯。

⌨️ 輸入控制系統

InputManager 維護每個按鍵的三種狀態： down（持續按住）、 justPressed（本幀剛按下）、 justReleased（本幀剛放開）。 每幀調用 input.update() 重置瞬態狀態。

Space ↑
跳躍 / 開始遊戲
按住 >0.2s 觸發 Boost，長壓可連跳（落地後若仍按住自動再跳）
↓ Ctrl
蜷縮 / 快速下墜
地面按住 → 蜷縮姿勢（碰撞箱縮短）；空中按住 → 快速下墜（重力×1.1/幀）
Q
Debug 加速
每按一次 +1 速度（僅 debug=true 時啟用）

Callback 系統

任何模組可透過 input.addKeyCallback() 監聽按鍵事件， 支援 maxCalls 限制（例如開始畫面只觸發一次）： 

// 只觸發一次的 callback（maxCalls=1）
game.setStarter(600);
// 內部：
input.addKeyCallback('space', 'justPressed', () => {
  game.start();
}, 1); // 第四參數 maxCalls=1

WebRTC DataChannel 控制對應

來自 Angular 前端的 P2P 指令，對應遊戲內的按鍵動作：

✨ 粒子特效（Three.js Nebula）

腳步塵土效果使用 three-nebula 粒子庫， 速度越快粒子越多越激烈，最高速度時完全停止（因為速度太快看不見）。 

// 建立腳步塵土 Emitter
function nebulaCreateDynoDustEmitter(spd = 5) {
  const emitter = new Nebula.Emitter();

  // 發射速率：每 0.1~0.25s 發射 1~2 個粒子
  emitter.rate = new Nebula.Rate(
    new Nebula.Span(1, 2),
    new Nebula.Span(0.1, 0.25)
  );

  // 初始化：質量 10、半徑 0.1、壽命 1~3 秒
  emitter.addInitializer(new Nebula.Radius(.1));
  emitter.addInitializer(new Nebula.Life(1, 3));

  // 行為：隨機旋轉、從大縮小（2→0.1）、框內邊界反彈
  emitter.addBehaviour(new Nebula.Rotate('random', 'random'));
  emitter.addBehaviour(new Nebula.Scale(2, 0.1));

  // 位置：恐龍腳下 (0, -1.1, 15.5)
  emitter.position.set(0, -1.1, 15.5);
}
速度等級粒子速度 (spd)狀態
開始 4 輕微塵土
Level 27 明顯塵土，重新建立 emitter
Level 310 激烈塵土，重新建立 emitter
Level 4停止emitter.dead=true，不再發射
💡
跳躍時 dynoDustEmitter.stopEmit()（離地）， 落地時 dynoDustEmitter.emit()（恢復）， 視覺上模擬腳著地瞬間揚起的塵土。
